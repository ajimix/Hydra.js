Hydra.js! Yes another modular architecture for your app!

History:

Hydra was a mithological creature (Multi-headed water serpent/dragon).
When one of their heads was cut off two grew back.

Description

A website without a modular system is prone to failure.
Hydra.js has come to solve this problem!

Designed to:

  • Avoid failing your full application for some little error.
  • Be extended.
  • Be framework agnostic.
  • Create a scalable, maintainable and module oriented system.

Install:

<script type="text/javascript" src="/path/to/Hydra.js"></script>

Hydra.js API

Hydra.action()

Returns the Action class.

Hydra.action().listen( aNotificationsToListen, fpHandler, oModule )
Listen adds new/s action/s to be listened by the module.
Params:
  • aNotificationsToListen : Array of id's of actions to listen.
  • fpHandler : Handler/Callback to manage the actions.
  • oModule : Module reference.
Hydra.action().notify( oNotifier )
Notify triggers the actions that are being listened by the module.
Params:
  • oNotifier : Notifier object.
    • oNotifier.type : Action id to be triggered.
    • oNotifier.data : Data that will be used when triggering the action.
Hydra.action().stopListen( aNotificationsToStopListen, oModule )
StopListen removes the indicated listeners for the module
Params:
  • aNotificationsToStopListen : Array of id's of actions to listen.
  • oModule : Module reference.

Hydra.errorHandler()

Returns the ErrorHandler class.

Hydra.errorHandler().log( )
Logs the error on the console if available. If console is not available the logs will be appended on a hidden layer.

Hydra.setErrorHandler( oErrorHandler )

SetErrorHandler can be used to change the ErrorHandler object to add more logging logic.
Params:
  • oErrorHandler : ErrorHandler to change the original. This object must have a "log" method at least.

Hydra.module

Module is the class that will manage the module system.

Hydra.module.register( sModuleId, fpCreator )
Register will add a new module to the module manager.
Params:
  • sModuleId : Id of the module to register.
  • fpCreator : Function that will return the object module when initialized.
Hydra.module.extend( sModuleId, fpCreator )
Extend is the method that must be used to extend a valid module.
Remember: The module to be extended must be registered before extend it.
Params:
  • sModuleId : Id of the module to be extended.
  • fpCreator : Function that will extend the base object module when initialized.
Hydra.module.start( sModuleId, oData )
Start is the method that will initialize the module.
Remember: When a module is started the "init" method is executed.
Params:
  • sModuleId : Id of the module to initialize.
  • oData : (Optional) This must be an object that will be supplied to the "init" method as first argument.
Hydra.module.startAll()
StartAll will initialize all the registered modules.
Hydra.module.stop( sModuleId )
Stop will stop/destroy the module.
Remember: No more actions listened by this module will be triggered again.
Params:
  • sModuleId : Id of the module to be stopped/destroyed.
Hydra.module.stopAll()
StopAll will stop/destroy all the registered modules.
Hydra.module.register( sModuleId, fpCreator )
Register will add a new module to the module manager.
Params:
  • sModuleId : Id of the module to register.
  • fpCreator : Function that will return the object module when initialized.
Hydra.module.test( sModuleId, fpCallback )
Test executes the callback with the instanced module as first parameter.
Params:
  • sModuleId : Id of the module to be returned.
  • fpCallback : Function that will be executed when the module is instanced.

Hydra.setDebug( bDebug )

SetDebug changes the debug flag.
Remember: True will activate the debug mode. This allows the modules to fail.
Remember: False will deactivate the debug mode. This avoid the modules to fail.
Params:
  • bDebug : Boolean value to activate/deactive debug mode.

Examples

Module:

Create a module:
                    	
	Hydra.module.register('moduleId', function(action)
	{
		return {
			init: function (oData) {},
			handleAction: function (oNotifier){
			},
			destroy: function () {}
		};
	});
Extend a module:
                    	
	Hydra.module.extend('moduleId', function(action)
	{
		return {
			init: function (oData) {},
			handleAction: function (oNotifier){
			},
			destroy: function () {}
		};
	});

Actions Listening:

To use the action manager you have it accessible inside the module as "action".

Tip: Use it on your init to start listening actions when the module starts.

Listen one action:
                    	
	Hydra.module.register('moduleId', function(action)
	{
		return {
			init: function (oData) {
				action.listen(['action1'], this.handleAction, this);
			},
			handleAction: function (oNotifier){
				if(oNotifier.type === 'action1')
				{
					/*your code*/
				}
			},
			destroy: function () {}
		};
	});
Listen two actions:
                    	
	Hydra.module.register('moduleId', function(action)
	{
		return {
			init: function (oData) {
				action.listen(['action1', 'action2'], this.handleAction, this);
			},
			handleAction: function (oNotifier){
				switch(oNofitier.type)
				{
					case 'action1':
						/* your code */
						break;
					case 'action2':
						/* your code */
						break;
					default: break;
				}
			},
			destroy: function () {}
		};
	});
Listen more actions:

Tip: If you have several actions to listen I recommend you to make use of an object where the keys are the names of the actions. 

                    	
	Hydra.module.register('moduleId', function(action)
	{
		return {
			init: function (oData) {
				action.listen(['action1', 'action2', 'actionN'], this.handleAction, this);
			},
			actionHandlers: {
				action1: function (oData) {},
				action2: function (oData) {},
				actionN: function (oData) {}
			},
			handleAction: function (oNotifier){
				var oHandler = this.actionHandlers[oNotifier.type]
				if(typeof oHandler === "undefined")
				{
					return;
				}
				oHandler.call(this, oData);
				oHandler = null;
			},
			destroy: function () {}
		};
	});

Actions Notifying:

To use the action manager you have it accessible inside the module as "action".

Tip: You can notify an action where ever you want inside the module using 'action'.

When notifying an action you will need to use a Notifier JSON object: 

								
	var oNotifier = {
		type: 'listenedAction',
		data: 'data'
	};

Notifying inside of a module:
                    	
	Hydra.module.register('moduleId', function(action)
	{
		return {
			init: function (oData) {
				$("#button").click(function(){
					action.notify({
						type: 'listenedAction',
						data: 'data'
					});
				});
			},
			handleAction: function (oNotifier){
			},
			destroy: function () {}
		};
	});
Notifying from outside of a module:
                    	
	var oAction = new (Hydra.action());
	oAction.notify({
		type: 'listenedAction',
		data: 'data'
	});
